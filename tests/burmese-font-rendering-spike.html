Burmese (Myanmar) Canvas Font Rendering Spike

Standalone test, not wired into the real app. Draws item-name text onto a canvas at the exact same dimensions and font size renderCompositeVoucher() in app.js uses for one voucher row (564×93px, bold 18px), with Noto Sans Myanmar added to the font stack ahead of Courier Prime so Myanmar codepoints resolve to a font that actually has those glyphs. If this looks right, the same font-stack change is safe to carry into the real renderer later for typed item names.

1. English only
2. Burmese only (includes a stacked/conjunct consonant -- a common Myanmar Unicode rendering stress point)
3. Mixed English + Burmese + Myanmar digits
4. Long Burmese item name (wrap/overflow check at real column width)
